<step number="2" name="yaml-frontmatter-generation">
<action>
1. In `crates/wtcd-mirror/src/template.rs`, implement YAML frontmatter generation:
   ```rust
   pub fn generate_header(
       file_result: &FileResult,
       fingerprints: &FingerprintResult,
       source_commit: &str,
       module_id: &str,
   ) -> MirrorHeader
   ```

2. Field derivation rules:
   - `anrsm_version`: `1`
   - `artifact_type`: `"file_mirror"`
   - `artifact_id`: `"file_mirror:<source_path>"`
   - `source_path`: from `FileResult.file_path`
   - `source_language`: derive from extension (`.ts`/`.tsx` → `"typescript"`, `.js`/`.jsx` → `"javascript"`)
   - `module_id`: derive from first path segment after source root (e.g., `src/auth/login.ts` → `"auth"`), fallback `"global"`
   - `source_commit`: passed as parameter (CLI will get from git)
   - `source_fingerprint`: from `FingerprintResult.source`
   - `semantic_fingerprint`: from `FingerprintResult.semantic`
   - `freshness_state`: `"fresh"` (always fresh on generation)
   - `confidence_band`: map from `FileResult.confidence` (`High` → `"high"`, `Low` → `"low"`, `None` → `"low"`)
   - `generator_name`: `"anrsm"`
   - `generator_version`: `"0.1.0"`
   - `generated_at`: current UTC time in RFC 3339 format
   - `exports`: `Some(vec![export.name for export in &file_result.exports])`
   - `dependencies`: `Some(vec![import.source for import in &file_result.imports])`
   - `risk_tags`: derive from side effects (e.g., `Network` → `"external_api"`, `Io` → `"file_io"`, `Storage` → `"persistence"`, `Log` → `"logging"`)
   - `manual_appendix_present`: `None` initially (set during I/O merge)

3. Implement YAML serialization:
   ```rust
   pub fn header_to_yaml(header: &MirrorHeader) -> Result<String>
   ```
   - Use `yaml_serde::to_string` to serialize header to YAML
   - Wrap in `---\n...\n---\n` frontmatter delimiters

4. Write unit tests:
   - Header field derivation from sample FileResult
   - YAML serialization produces parseable output
   - `artifact_id` format matches `"file_mirror:..."` pattern
   - `source_fingerprint` format matches `"sha256:..."` pattern

<step number="3" name="mirror-body-generation">
<action>
1. In `crates/wtcd-mirror/src/template.rs`, implement 8-section body generation (D-01, D-02):
   ```rust
   pub fn generate_body(file_result: &FileResult) -> MirrorBody
   ```

2. Section generation rules (D-02: rule-based, structured):

   **Section 1: 职责 (Responsibilities)**
   - D-03: Derive from exported symbols + file path
   - If exports exist: `"暴露 {symbol_names}；{file_path_hint}"`
   - If no exports: `"内部实现模块，不对外暴露符号。"`
   - File path hint: last path segment without extension (e.g., `login.ts` → "登录相关逻辑")

   **Section 2: 对外契约 (External Contracts)**
   - List exported function signatures with parameter types and return type
   - Format: `"对外暴露 {name}({params}) → {return_type}"`
   - If no signatures: `"无显式函数契约。"`

   **Section 3: 输入与前置条件 (Preconditions)**
   - List import dependencies as preconditions
   - Format: `"依赖 {import_sources} 提供的符号和类型。"`
   - If no imports: `"无外部依赖。"`

   **Section 4: 状态与控制流 (State/Control Flow)**
   - Describe main exported functions and their flow
   - For each signature: `"{name}({params}): {description_based_on_side_effects}"`
   - If `confidence == Low`: `"（解析置信度低，可能遗漏分支。）"`

   **Section 5: 副作用与外部依赖 (Side Effects)**
   - List side effects by kind
   - Format: `"{Kind}: {target} (行 {line})"`
   - Kinds: Io → "文件I/O", Network → "网络请求", Storage → "存储操作", Log → "日志记录"
   - If no side effects: `"无检测到的副作用。"`

   **Section 6: 关键不变量与风险 (Invariants/Risks)**
   - Infer from exports + side effects
   - If has Network/Storage side effects: `"涉及外部系统调用，需关注错误处理和幂等性。"`
   - If has exports + side effects: `"导出符号包含副作用调用方，调用方需了解副作用边界。"`
   - If confidence == Low: `"解析置信度低，不变量可能不完整。"`

   **Section 7: 变更影响 (Change Impact)**
   - D-04: Derive from import dependency graph
   - List files that import this file's exports (reverse dependency)
   - If no reverse deps available: list the file's own imports as "可能受上游变更影响"
   - Format: `"变更可能影响: {affected_files}"`

   **Section 8: 何时必须展开源码 (Expand Conditions)**
   - Generic rules:
     - If has side effects: `"修改副作用处理逻辑时必须展开源码。"`
     - If has exported signatures: `"修改函数签名或返回类型时必须展开源码。"`
     - If confidence == Low: `"解析置信度低，关键决策前必须展开源码。"`
     - Always: `"涉及错误处理、并发控制或状态管理时必须展开源码。"`

3. Implement body serialization:
   ```rust
   pub fn body_to_markdown(body: &MirrorBody) -> String
   ```
   - Format: `## {section_title}\n{section_content}\n`

4. Write unit tests:
   - Body has exactly 8 sections
   - Section titles match spec (职责, 对外契约, etc.)
   - Exports appear in 职责 section
   - Side effects appear in 副作用 section
   - Imports appear in 前置条件 section
   - Empty file produces valid body (no panics)
</action>
