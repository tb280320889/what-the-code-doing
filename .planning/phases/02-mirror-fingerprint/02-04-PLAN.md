<objective>
Implement mirror file I/O: reading existing mirrors, writing with manual_appendix preservation, and orphan cleanup when source files are deleted.
</objective>

<implementation>
<step number="1" name="mirror-writing">
<action>
1. Create `crates/wtcd-mirror/src/io.rs` — implement mirror file writing:
   ```rust
   /// Write a MirrorFile to disk at the given mirror path.
   /// Creates parent directories as needed.
   pub fn write_mirror(mirror_path: &Path, content: &str) -> Result<()>
   ```

2. Implement `write_mirror_with_appendix`:
   ```rust
   /// Write mirror content, preserving any existing manual_appendix between markers.
   /// 
   /// If the mirror file already exists:
   ///   1. Read existing file
   ///   2. Extract manual_appendix between <!-- manual_appendix_start --> and <!-- manual_appendix_end -->
   ///   3. Write new content with appendix injected after the last standard section
   /// If no existing file or no appendix markers: write content as-is.
   pub fn write_mirror_with_appendix(
       mirror_path: &Path,
       new_content: &str,
       preserve_appendix: bool,
   ) -> Result<AppendixResult>
   ```

3. Define `AppendixResult`:
   ```rust
   pub struct AppendixResult {
       pub appendix_preserved: bool,
       pub appendix_content: Option<String>,
   }
   ```

4. Appendix handling rules (D-10):
   - Markers: `<!-- manual_appendix_start -->` and `<!-- manual_appendix_end -->`
   - If `preserve_appendix == false`: skip appendix extraction, overwrite entirely
   - If existing file has markers between them: extract content, append after body
   - If existing file has no markers: write new content as-is
   - If new_content already has markers (from previous generation): replace with extracted content

5. Write unit tests:
   - Write new file → file created with correct content
   - Write to existing file without appendix → overwrites
   - Write to existing file with appendix → preserves appendix content
   - Write with `preserve_appendix == false` → overwrites appendix
   - Parent directories created automatically
</action>

<step number="3" name="orphan-cleanup">
<action>
1. In `crates/wtcd-mirror/src/io.rs`, implement orphan detection:
   ```rust
   /// Given a list of current source file paths and a mirror root directory,
   /// return mirror file paths that have no corresponding source file.
   /// These are "orphan" mirrors that should be deleted (D-12).
   pub fn find_orphan_mirrors(
       source_paths: &[String],
       mirror_root: &Path,
   ) -> Result<Vec<PathBuf>>
   ```

2. Implementation:
   - Walk mirror_root directory for all `.md` files
   - For each `.md` file, compute expected source path using `mirror_to_source_path`
   - If source path not in `source_paths` list → mark as orphan
   - Return list of orphan paths

3. Implement `delete_orphans`:
   ```rust
   /// Delete orphan mirror files (D-12).
   /// Returns count of deleted files.
   pub fn delete_orphans(orphans: &[PathBuf]) -> Result<usize>
   ```

4. Write unit tests:
   - Setup: create temp dir with mirror files, some with matching sources, some orphaned
   - `find_orphan_mirrors` returns only orphaned paths
   - `delete_orphans` removes files, returns correct count
   - No orphans → empty result
   - Empty mirror root → empty result
