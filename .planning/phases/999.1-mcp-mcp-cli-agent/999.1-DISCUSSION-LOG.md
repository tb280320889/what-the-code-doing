## MCP 与 CLI 的关系

| Option | Description | Selected |
|--------|-------------|----------|
| MCP 主入口，CLI 辅助 | MCP 是主要 Agent 集成入口，CLI 保留为人类和 CI 使用 | |
| 完全平等共存 | CLI 和 MCP 共享核心库，各自独立交付层 | ✓ |
| MCP 是 CLI 的超集 | MCP 实验性额外入口，CLI 仍是正式 Agent 集成方式 | |
| MCP 完全替代 CLI | 废弃 CLI 驱动 Agent 集成，全面转向 MCP | |

**User's choice:** 完全平等共存

**Core engine interaction:**

| Option | Description | Selected |
|--------|-------------|----------|
| 直接调用 Rust API | MCP Server 调用 wtcd-core 等 crate 的 Rust API，无序列化开销 | ✓ |
| 调用 CLI 二进制 | MCP Server 通过 stdio 调用 wtcd-cli，解析 JSON 输出 | |
| 混合模式 | 高效操作直接调用 API，简单查询调用 CLI | |

**User's choice:** 直接调用 Rust API

**Compatibility:**

| Option | Description | Selected |
|--------|-------------|----------|
| Claude Code + Cursor 优先 | 最直接的集成路径 | |
| 覆盖主流 AI Agent | 支持更多 Agent，需要更广泛传输 | |
| 协议级通用兼容 | 任何支持 MCP 协议的 Agent 都能用 | ✓ |

**User's choice:** 协议级通用兼容（但 stdio-only，不提供 HTTP 后端服务）

**Spec handling:**

| Option | Description | Selected |
|--------|-------------|----------|
| 需要新规范文档 | 新增 MCP 工具定义和资源暴露规范 | ✓ |
| 扩展现有规范 | 修改 docs/08 增加 MCP | |
| 先 PoC 后规范 | 先验证可行性再补规范 | |

**User's choice:** 需要新规范文档

**Notes:** 用户后续补充：所有设计按本地 stdio 来选，不提供后端服务功能。

## Agent 发现与连接方式

**Discovery:**

| Option | Description | Selected |
|--------|-------------|----------|
| 手动配置 | 用户手动在 Agent 配置中添加 MCP Server 路径 | |
| init 自动生成配置片段 | wtcd init 生成 Agent 特定配置片段 | |
| 专用安装命令 | wtcd mcp-install --agent claude，自动写入配置 | ✓ |

**User's choice:** 专用安装命令

**Binary strategy:** Agent 决定 → 单一二进制含 MCP 模式（最低 UX 心智负担）

**Lifecycle:** Agent 决定 → 预加载 + 按需（最适合本地 stdio）

**Project switching:**

| Option | Description | Selected |
|--------|-------------|----------|
| 单项目 per 实例 | 不同项目需要不同 MCP Server 实例 | |
| 单实例多项目 | 一个实例服务多个项目，首次访问时自动加载 | ✓ |
| 先单项目，后多项目 | 先实现单项目，多项目留到 HTTP 阶段 | |

**User's choice:** 单实例多项目
