# Phase 999.1 Research: MCP Server for WTCD

**Researched:** 2026-03-21
**Confidence:** HIGH

## Executive Summary

WTCD 应使用 **rmcp crate**（MCP 协议的官方 Rust SDK）构建 MCP Server，通过 **stdio 传输**暴露 4 个核心工具给 Agent（Claude Code、Cursor 等）。rmcp 提供宏驱动的工具定义系统（`#[tool]`、`#[tool_router]`、`#[tool_handler]`），与现有 WTCD crate 无缝集成。

### 核心 API 模式

#### 工具定义（#[tool] 宏）

```rust
use rmcp::{
    handler::server::tool::ToolRouter,
    model::{CallToolResult, Content, ServerInfo, ServerCapabilities},
    tool, tool_handler, tool_router,
    ErrorData as McpError, ServiceExt,
    transport::stdio,
};

#[derive(Clone)]
pub struct WtcdMcpServer {
    tool_router: ToolRouter<Self>,
}

#[tool_router]
impl WtcdMcpServer {
    pub fn new() -> Self {
        Self {
            tool_router: Self::tool_router(),
        }
    }

    #[tool(description = "Parse TS/JS files and extract semantic facts")]
    async fn wtcd_parse(
        &self,
        Parameters(params): Parameters<ParseRequest>,
    ) -> Result<CallToolResult, McpError> {
        // 调用 wtcd-core/wtcd-adapters 的 Rust API
        Ok(CallToolResult::success(vec![Content::text(result_json)]))
    }
}
```

#### 工具参数（Parameters + schemars）

```rust
use rmcp::handler::server::tool::Parameters;
use schemars::JsonSchema;
use serde::{Deserialize, Serialize};

#[derive(Debug, Deserialize, Serialize, JsonSchema)]
pub struct ParseRequest {
    /// 要解析的文件路径（相对项目根目录）
    pub file_path: String,
    /// 可选：指定语言适配器（auto 为自动检测）
    #[serde(default = "default_language")]
    pub language: String,
}
```

#### ServerHandler 实现

```rust
#[tool_handler]
impl ServerHandler for WtcdMcpServer {
    fn get_info(&self) -> ServerInfo {
        ServerInfo {
            protocol_version: ProtocolVersion::V_2025_03_26,
            instructions: Some(
                "WTCD MCP Server — 提供代码语义解析、路由、检查和镜像生成工具"
            ),
            capabilities: ServerCapabilities::builder().enable_tools().build(),
            server_info: Implementation {
                name: "wtcd-mcp".to_string(),
                version: env!("CARGO_PKG_VERSION").to_string(),
            },
        }
    }
}
```

#### 入口点（stdio 传输）

```rust
use rmcp::{ServiceExt, transport::stdio};

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    tracing_subscriber::fmt()
        .with_writer(std::io::stderr)  // 日志写 stderr，stdout 保留给 JSON-RPC
        .with_env_filter(EnvFilter::from_default_env())
        .init();

    let server = WtcdMcpServer::new();
    let service = server.serve(stdio()).await.inspect_err(|e| {
        tracing::error!("MCP server error: {}", e);
    })?;

    service.waiting().await?;
    Ok(())
}
```

## MCP Tool 设计

### 4 个核心工具

| 工具名 | 功能 | 对应 wtcd-* crate | MVP 状态 |
|--------|------|-------------------|----------|
| `wtcd_parse` | 解析文件，提取语义事实 | wtcd-core + wtcd-adapters | ✅ 实现 |
| `wtcd_run` | 扫描项目，输出全量解析结果 | wtcd-core + wtcd-scope + wtcd-adapters | ✅ 实现 |
| `wtcd_check` | 检测镜像漂移 | Phase 3 能力 | ⏸ 注册但 stub |
| `wtcd_mirror` | 生成镜像文件 | Phase 2 能力 | ⏸ 注册但 stub |

## 关键技术挑战

### 1. 同步 → 异步桥接

现有 wtcd-core/wtcd-adapters/wtcd-scope 全部是同步代码。MCP Server 需要 tokio async runtime。

**解决方案：** 使用 `tokio::task::spawn_blocking()` 包装同步调用：

```rust
#[tool(description = "Parse a TS/JS file")]
async fn wtcd_parse(&self, params: Parameters<ParseRequest>) -> Result<CallToolResult, McpError> {
    let result = tokio::task::spawn_blocking(move || {
        // 同步调用 wtcd-adapters
        wtcd_adapters::parse_file(&file_path)
    }).await.map_err(|e| McpError::internal_error(e.to_string(), None))?;
    
    Ok(CallToolResult::success(vec![Content::text(
        serde_json::to_string(&result)?
    )]))
}
```

### 2. 配置发现

MCP Server 启动时的 cwd 未必是项目根目录。

**解决方案：**
1. 工具参数中接受 `project_root` 参数
2. 首次工具调用时自动查找最近的 `anrsm.yaml`
3. 缓存已加载的配置（HashMap<String, Config>）

### 3. 生命周期管理

stdio 模式下，MCP Server 作为 Agent 的子进程运行，由 Agent 管理生命周期。

**约束：**
- 不能有长时间运行的后台任务
- 不能持有数据库连接（用完即关）
- 日志写 stderr（stdout 保留给 JSON-RPC）

### 4. MSRV 对齐

| 依赖 | MSRV | 兼容性 |
|------|------|--------|
| 项目当前 MSRV | 1.85 | — |
| rmcp 0.16.0 | 需验证 | ⚠️ |
| tokio 1.x | 1.70+ | ✅ |
| schemars 0.8 | 1.60+ | ✅ |

**Action:** 需要在构建时验证 rmcp 0.16.0 是否与 MSRV 1.85 兼容。如果不兼容，可能需要降低 MSRV 要求或使用 rmcp 0.15.x。

## 不做的事

| 避免 | 原因 |
|------|------|
| 自己实现 JSON-RPC | rmcp 完整处理协议层，不要重复造轮子 |
| HTTP/SSE 传输 | D-03 锁定仅 stdio，MVP 不做远程 |
| MCP Resources | D-07 锁定 MVP 仅 Tools，Resources 未来考虑 |
| MCP Prompts | 有价值但不在 MVP 范围 |
| 动态工具注册 | 编译时确定工具列表，运行时不变 |
