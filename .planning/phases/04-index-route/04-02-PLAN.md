# Plan 04-02: Query Engine — Scoring + Ranking

## Objective

Implement the keyword-based relevance scoring and ranking engine for routing index queries.

**Purpose:** Core algorithm that converts "modify auth logic" into ranked file candidates with relevance scores.
**Output:** `route_query()` function in `crates/wtcd-core/src/index.rs`.

## Context

@crates/wtcd-core/src/index.rs

Decisions to implement: D-05 (keyword matching on exports + source_path + module_id), D-06 (freshness weights), D-07 (score = hits/query_tokens × freshness_weight), D-08 (tie-break: confidence desc, source_path alpha).

    1. **RouteResult struct** — per D-16 (JSON output format):

       pub struct RouteResult {
           pub source_path: String,
           pub artifact_id: String,
           pub relevance_score: f64,
           pub freshness: String,
           pub exports: Vec<String>,
           pub confidence: String,
       }

    2. **RouteOutput struct** — top-level CLI output:
       ```rust
       #[derive(Debug, Clone, Serialize, Deserialize)]
       pub struct RouteOutput {
           pub api_version: String,
           pub query: String,
           pub total_candidates: usize,
           pub results: Vec<RouteResult>,
       }
       ```

    3. **freshness_weight function** — per D-06:
       ```rust
       fn freshness_weight(freshness: &str) -> f64 {
           match freshness {
               "fresh" => 1.0,
               "stale" => 0.7,
               "invalid" => 0.3,
               _ => 0.5, // unknown
           }
       }
       ```

    4. **tokenize_query function**:
       - Lowercase the query
       - Split by whitespace and common punctuation (._-/)
       - Filter out empty strings
       - Return Vec<String>

       Example: "modify auth logic" → ["modify", "auth", "logic"]
       Example: "update user profile.ts" → ["update", "user", "profile", "ts"]

    5. **route_query function** — per D-07, D-08:
       ```rust
       pub fn route_query(query: &str, index: &RoutingIndex, top_k: usize) -> RouteOutput
       ```
       Algorithm:
       - Tokenize query → Vec<token>
       - For each entry in index.entries:
         - Count how many query tokens match entry.keywords (case-insensitive)
         - If matches == 0, skip entry
         - Score = matches as f64 / query_tokens.len() as f64 × freshness_weight(&entry.freshness)
         - Build RouteResult with score
       - Sort results by:
         - relevance_score descending
         - confidence descending (map high→2, low→1, none→0 for comparison)
         - source_path ascending (alphabetical)
       - Take top_k
       - Return RouteOutput { api_version: "1", query, total_candidates: results.len(), results }

    6. Write tests:
       - route_query exact match: query "login" matches entry with keyword "login"
       - route_query partial match: query "auth login" matches entry with keywords ["auth", "login"] → score 1.0
       - route_query no match returns empty results
       - freshness_weight affects score: stale entry scores lower than fresh with same keyword match
       - tie-break: two entries with same score, different confidence → higher confidence first
       - tie-break: same score, same confidence → alphabetical source_path
       - top_k limits results
       - top_k=0 returns empty
       - case insensitivity: query "Auth" matches keyword "auth"
       - compound keyword match: query "login" also matches keyword extracted from "userLogin" camelCase

    7. Update `crates/wtcd-core/src/lib.rs` if needed (exports)
