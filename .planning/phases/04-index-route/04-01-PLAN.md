# Plan 04-01: Routing Index Types + Builder

## Objective

Create the routing index data model and builder that transforms `Vec<FileResult>` into a serializable routing index.

**Purpose:** Foundation for the route command — all downstream query/ranking depends on this index structure.
**Output:** `crates/wtcd-core/src/index.rs` with types, keyword extraction, and index I/O.

## Context

@crates/wtcd-core/src/types.rs
@crates/wtcd-core/src/depgraph.rs
@crates/wtcd-core/src/lib.rs
@crates/wtcd-mirror/src/template.rs

Existing types to reuse:
- `FileResult` — source data for building index entries
- `ExportedSymbol` — export names become keywords
- `DependencyEdge` — dependency sources stored in index
- `ConfidenceBand` — maps to index confidence field
- `derive_risk_tags()` pattern in `wtcd-mirror/src/template.rs` — side effects → risk tags

Decisions to implement: D-01 (JSON file), D-02 (flat array format), D-03 (auto-generated by run), D-04 (only mirrored files), D-09 (keyword sources), D-10 (risk_tags), D-11 (dependencies), D-12 (from FileResult).

  <name>Task 1: Define RoutingIndexEntry and RoutingIndex types</name>
  <files>crates/wtcd-core/src/index.rs</files>
  <behavior>
    - RoutingIndexEntry has fields: artifact_id, module_id, source_path, exports: Vec<String>, keywords: Vec<String>, dependencies: Vec<String>, risk_tags: Vec<String>, freshness, confidence
    - RoutingIndex has fields: api_version: "1", entries: Vec<RoutingIndexEntry>
    - Both derive Serialize, Deserialize, Debug, Clone
    - freshness is String (fresh/stale/invalid/unknown)
    - confidence is String (high/low/none)
  </behavior>
  <action>
    Create `crates/wtcd-core/src/index.rs` with:

    1. **RoutingIndexEntry struct** — per D-02:
       ```rust
       #[derive(Debug, Clone, Serialize, Deserialize)]
       pub struct RoutingIndexEntry {
           pub artifact_id: String,
           pub module_id: String,
           pub source_path: String,
           pub exports: Vec<String>,
           pub keywords: Vec<String>,
           pub dependencies: Vec<String>,
           pub risk_tags: Vec<String>,
           pub freshness: String,
           pub confidence: String,
       }
       ```

    2. **RoutingIndex struct** — top-level wrapper:
       ```rust
       #[derive(Debug, Clone, Serialize, Deserialize)]
       pub struct RoutingIndex {
           pub api_version: String,
           pub entries: Vec<RoutingIndexEntry>,
       }
       ```

    3. **extract_keywords function** — per D-09:
       - Input: source_path, exports: &[String], module_id: &str
       - Split source_path by `/`, then each segment by `_` (snake_case) and by uppercase boundaries (camelCase)
       - Push all export names directly as keywords
       - Push module_id as keyword
       - Deduplicate and lowercase all keywords
       - Return Vec<String>

       Example: source_path="src/auth/login.ts", exports=["login","logout"], module_id="auth"
       → keywords: ["src", "auth", "login", "logout"]

       CamelCase splitting: "parseAuthTokens" → ["parse", "auth", "tokens"]

    4. **derive_risk_tags function** — mirror the pattern from wtcd-mirror/src/template.rs:
       - Map SideEffectKind::Network → "external_api"
       - Map SideEffectKind::Io → "file_io"
       - Map SideEffectKind::Storage → "persistence"
       - Map SideEffectKind::Log → "logging"
       - Deduplicate and sort

    5. **build_routing_index function** — per D-12:
       - Input: file_results: &[FileResult], module_ids: &HashMap<String, String> (source_path → module_id)
       - Filter out files with ConfidenceBand::None (D-04: only mirrored files)
       - For each FileResult, build RoutingIndexEntry:
         - artifact_id = format!("file_mirror:{}", file_result.file_path)
         - module_id from module_ids map (default to "global")
         - source_path = file_result.file_path
         - exports = file_result.exports.iter().map(|e| e.name).collect()
         - keywords = extract_keywords(...)
         - dependencies = file_result.imports.iter().map(|i| i.source).collect()
         - risk_tags = derive_risk_tags(&file_result.side_effects)
         - freshness = "fresh" (default for newly built index)
         - confidence = match file_result.confidence { High → "high", Low → "low", None → "none" }
       - Return RoutingIndex { api_version: "1", entries }

    6. **write_routing_index function** — per D-01:
       - Input: index: &RoutingIndex, mirror_root: &Path
       - Serialize to pretty JSON
       - Write to mirror_root.parent().join("routing_index.json")  (sibling of mirror/file/)
       - Actually, per D-01 it's `mirror/routing_index.json`, so write to mirror_root.join("../routing_index.json") or better: accept the index path directly

    7. **read_routing_index function**:
       - Input: path: &Path
       - Read file, deserialize as RoutingIndex
       - Return Result<RoutingIndex>

    8. Add `pub mod index;` to `crates/wtcd-core/src/lib.rs`

    9. Write tests:
       - extract_keywords with snake_case paths
       - extract_keywords with camelCase exports
       - extract_keywords with nested paths (src/auth/login.ts)
       - extract_keywords deduplicates (export name same as path component)
       - derive_risk_tags maps correctly
       - build_routing_index from sample FileResult
       - build_routing_index filters out ConfidenceBand::None
       - write/read round-trip with tempfile
