# Phase 04 Plan Index + Route Summary

## One-liner
Keyword-based routing index with relevance scoring — transforms "modify auth logic" into ranked file candidates with freshness weighting.

## What Was Built

### Routing Index Module (`crates/wtcd-core/src/index.rs`)
- **RoutingIndexEntry** — per-entry data: artifact_id, module_id, source_path, exports, keywords, dependencies, risk_tags, freshness, confidence
- **RoutingIndex** — top-level wrapper with api_version and entries array
- **extract_keywords** — camelCase + snake_case splitting from source_path, exports, and module_id
- **derive_risk_tags** — side effects → risk tag mapping (mirrors wtcd-mirror pattern)
- **build_routing_index** — transforms `Vec<FileResult>` → `RoutingIndex`, filters ConfidenceBand::None
- **write_routing_index / read_routing_index** — JSON I/O
- **route_query** — keyword-based scoring with freshness weighting and tie-breaking
- 23 unit tests covering all functions

### Query Engine
- **Scoring formula**: `matching_keyword_count / query_token_count × freshness_weight`
- **Freshness weights**: fresh=1.0, stale=0.7, invalid=0.3, unknown=0.5
- **Tie-breaking**: confidence descending (high>low>none), then source_path alphabetical
- **Case-insensitive** keyword matching
- **top_k** parameter limits results

### CLI Integration
- **`wtcd route "query"`** — new subcommand with `--top-k` flag (default 10)
- **`wtcd run`** — auto-rebuilds `mirror/routing_index.json` after mirror generation
- Missing index produces clear error: "Run 'wtcd run' first"

### Integration Tests
- 9 integration tests in `crates/wtcd-tests/tests/integration_route.rs`
- Covers: index building, query ranking, freshness scoring, I/O round-trip, missing index, JSON serialization
